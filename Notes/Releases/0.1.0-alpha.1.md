# Withdrawn Alpha 0.1.0-alpha.1

This release has been withdrawn and should not be used for future friend-facing playtests.

The main reason is that it mixed real project mechanics with borrowed vanilla mechanics in the traveling fire spell path, which made the playtest misleading.

## What this release represented

- `glyph_focus` item for player-facing casting
- Glyph Composer screen opened with `G` while holding the focus
- Quick-cast for the last composed chain with `R`
- Always-visible mana HUD with segmented mana pips
- HUD spell feedback for cast and analyze events
- Physical locks, linked keys, and magical lock-state tags
- Prototype spells for force, healing, vitality transfer, unlocking, wards, and simple construction
- `stone_path` now replaces targeted ground surfaces instead of floating above them

## Why it was withdrawn

- Traveling fire effects were still inheriting vanilla `LargeFireball` behavior instead of using owned spell runtime logic.
- The release line is being replaced by `dev-release` builds rather than continued alpha tags.

## Original playtest goals

- Check whether composing spells through the focus feels understandable without chat commands
- See whether the current mana pacing is readable enough during real fights and movement
- Try lock/key interactions on doors, trapdoors, fence gates, and chests
- Stress-test multiplayer ward triggers and recent spell feedback visibility
- Find ambiguity cases where the interpreter resolves to surprising outcomes

## Known limitations

- The focus composer still reuses command-backed casting under the hood
- Input is still button-driven rather than the future timing and chord system
- Final HUD art, particles, and spell visuals are still using placeholder presentation
- Construction, healing, and lock interactions are still prototype slices rather than final balance
